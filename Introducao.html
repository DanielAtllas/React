React - Introdução

Basicamente, a primeira dúvida do desenvolvedor está em decidir entre quais dos reacts escolher para iniciar seus estudos, react.js ou react native. A grande questão, é que em tese e de modo geral, os dois possuem uma mesma lógica e sintaxe semelhante na criação de seus códigos, se difereindo apenas de nomeclaturas e objetivo, uma vez que o react js possui foco em aplicações web e o react native com foco em desenvolvimento mobile.

    • Biblioteca
    • Aplicações web

  • React Native

Diante disso, focaremos nossos estudos no react js, de forma a entendermos melhor seu processo de desenvolvimento. Portanto, comecemos com a grande pergunta:

O que é React?

Como mencionado, o react é uma biblioteca utilizada para criar interfaces para usuário. Além disso, devido ter sua sintaxe clara, pode ser utilizado em diferentes aplicações como: web, mobile e etc. Dessa forma, sua grande característica está no uso de componentes, ou seja, seu desenvolvimento é separado em partes que irão conter na aplicação, como no exemplo abaixo:

Podemos dizer, que o react de modo geral, nada mais é do que uma extensão/biblioteca javaScript, devido a sua sintaxe e implementação estar diretamente relacionada a códigos .js. Dessa forma, para iniciar suas implementações, o desenvolviedor deverá ter em sua máquina instalado o Node Js, sendo a extensão necessária para a criação de programas na linguagem javaScript.

Start do servidor

Tendo entendido o conceito geral do que é o react e para que ele serve, podemos partir para a prática e começarmos as implementações de nosso primeiro programa. Como mencionado anteriormente, existe a necessidade do desenvolvedor ter instalado em sua máquina o Node Js para que a aplicação javaScript possa rodar e ser executada. Após isso, tendo definido o diretório onde a aplicação ficará armazenada, deveremos abrir o console/cmd/shell e darmos o seguinte comando:

npx create-react-app .

Este comando tem como função facilitar o processo de criação de aplicações em react, sendo desenvolvido pelo Facebook afim de cooperar com seus desenvolvedores na criação de novos programas. Dessa forma, esta extensão dará ao usuário as principais necessidades utilizadas em um projeto react para o desenvolvedor!

Após a instalação desta extensão estar completa, poderá ser percebido a criação automática de alguns arquivos no diretório em questão, os quais serão necessários para o projeto. Entre eles, vale destacar o arquivo package.json, o qual é necessário para todo e qualquer projeto que utiliza do Node Js. Este contém as configurações necessárias para rodar e executar o projeto react. Para termos certeza de que os primeiros passos foram implementados correntamente, podemos iniciar logo de cara o programa (mesmo que não tenha sido implementado nada), onde será exibido a interface que foi carregada junto a extensão mencionada no comando anteriormente. Utilizamos então o comando:

npm start

Podemos perceber então, que nosso projeto está criado em sua forma primária e podendo ser executado sem nenhum erro. Após isso, vá no diretório public e veja o arquivo de nome index. Neste arquivo, podemos perceber que seu corpo foi implementando utilizando recursos de html e css. Além disso, no body do arquivo podemos ver a tag de criação de uma div com o id de root (neste arquivo, está implementada a primeira interface da aplicação, ou seja, caso o usuário deseje implementar algo que altere o fundo da tela, deverá ser alterado aqui no body). Esta div, faz uma conexão com outro arquivo (arquivo /src/index.js) que fornece a este, as configurações necessárias para um projeto react, inserindo os componentes necessários, renderizando-os e aplicando no projeto react.

Alguns outros arquivos podem, no momento, serem julgados como desncessários, tendo apenas algumas funcionalidades de sistema que não alteram a rotina principal da aplicação. Portanto, por medidas de capricho e eliminação de lixo, podemos deletar os seguintes arquivos:

  • App.test.js

  • logo.svg

  • reportWebVit

  • setupTests.js

Após isso, por termos deletados tais arquivos, deveremos retirar sua importação realizada no arquivo index.js e retirar todo o conteúdo escrito a partir da linha 13 até a linha 18. Além disso, podemos ir até o arquivo App.css e deletar todos os comandos lá escritos (isto, porque tudo aquilo será um lixo para o programador e podemos utilizar este campo para estilizarmos nossa aplicação da forma que desejarmos).

Por se tratar de uma aplicação React, algumas funcionalidades são exclusivas desta linguagem. Portanto, altere o nome do arquivo /src/App.js para /src/App.jsx (a clausula de definição para arquivos react, sendo .jsx). Açém disso, todos os componentes criados, devem seguir esta mesma ideia e portanto, a identificação .jsx

Após isso, podemos entrar no arquivo de nome /src/App.js, o qual é o arquivo que contém o nosso primeiro componente criado, com o nome de function app().

Basicamente, é neste arquivo em que focaremos nossas primeiras implementações e por isso, a primeira coisa se fazer, é apagar todos os comandos do arquivo! Exatamente, apague todos os comandos para que possamos criar nossa aplicação do zero e assim inserirmos apenas os comandos que julgarmos necessários.

A partir deste ponto, todos os arquivos da aplicação estão em sua forma primitiva e portanto, devemos molda-los para criar nossa primeira aplicação!
