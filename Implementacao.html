React - Implementação

Bom, a partir deste ponto, temos compreendido toda a estrutura e funcionamento base do React e agora, podemos partir para a nossa implementação dos códigos e finalmente darmos forma ao nosso desenvolvimento. O primeiro comando a ser dado no arquivo, será o de importação do react, através do comando:

import React from 'react'

Todos os arquivos react de nossa aplicação, devem ter esta importação para seu funcionamento, sejam eles componentes ou não.

Este comando faz com que todos os códigos implementados abaixo deste, sejam interpretados pelo arquivo como códigos react e assim aplicados.

Em relação aos demais arquivos,

Após, podemos criar novamente o nosso bloco de componente, seguindo por exemplo, o comando:

        
        const App = () => {
            return(
                <>
                    <h1> Hello World! </h1>
                </>
            )
        }
        
        export default App;

Explicando melhor este pequeno pedaço de código, temos os seguintes comandos:

  • const App - Comando para criação do componente de nome App

  • return() - Campo de implementação do código que irá compor o componente em questão, de forma que o componente retorne ao arquivo index os comandos nele implementados.

  • export default App; - Comando de exportação deste componente para arquivos terceiros (serve como uma permissão de uso deste para demais arquivos).

OBS: Como dica, saiba que ao inserir o comando imr e o comando sfc e aprtamos a tecla de tab/complemento, estes comandos principais serão auto implementados no código.

Vale lembrar que o react possui a possibilidade de utilizar comandos semelhantes ao javaScript e por isso, podemos abusar deste conhecimento para implementações em nosso código, como por exemplo o usu de variáveis, através do comando const. Como na descrição exemplar abaixo:

        const App = () => {
            let message = "Hello World";
        
            return(
                <>
                    <h1> {message} </h1>
                </>
            )
        }
        
        export default App;

Porém, existe algo que deve ser comentado para os desenvolvedores react, pois nas linguagens de programação comuns, temos o hábito de declar uma variável e seu valor (como no exemplo acima com a variável message) e durante o percorrer do código, alterarmos este valor quando julgarmos necessário. Porém, no react este tipo de trabalho requer uma diferente imlementação para variável, devendo assim declará-la como state, ou seja, através desta declaração podemos alterar o valor de uma variável sempre que julgarmos necessário, fazendo com que a aplicação react reaja e renderize o seu novo valor.

Para isso, segue abaixo um exemplo de como utilizarmos deste recurso:

        const App = () => {
            const [message, setMessage] = useState("Hello World");
        
            return(

                <>
                    <div className="container"> {message} </div>
                    
                    <button onClick={() => setMessage("Olá Mundo!")}>
                        Mudar Mensagem
                    </button>
                </>
            )
        }
        
        export default App;

Basicamente, em sua declaração devemos passar o nome da variável (message) e também a função que terá o poder de alterar seu valor (setMessage). Em seguida, é usado o comando useState() para atribui-la a este recurso (vale lembrar que ao utilizar deste comando, deverá automaticamente importar a classe de mesmo nome).

Outra questão que vale a pena ressaltar, é o fato de podermos importar arquivos para o nosso programa react, como por exemplo a importação de um arquivo .css (por utilizar uma implementação de código semelhante ao HTML, o react também suporta códigos e arquivos do tipo style da linguagem CSS). Para isso, utilizamos do comando import, como no exemplo a seguir:

import "./App.css"

Componentes

Como mencionado, os componentes são a principal base de uso do React, os quais ao serem configurados, dão forma a implementação e por isso, eles funcionam de forma separada, ou seja, cada componente tem sua função distinta na aplicação, mas que coopera com os demais implementados. Diante disso, podemos deduzir que requer um certo domínio de organização para com o seu desenvolvimento. Então, o recomendado a ser feito, é criar um diretório em que os arquivos dos componentes serão armazenados, como por exemplo um diretório src/components.
